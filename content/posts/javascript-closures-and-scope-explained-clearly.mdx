## What are Closures?

A closure is a function that has access to its own scope and the scope of its outer functions. This means that a closure can use variables from its own scope and from the scope of any outer functions, even when the outer function has returned. To illustrate this, let's consider an example:

```javascript
function outer() {
  let count = 0;

  function inner() {
    count++;
    console.log(count);
  }

  return inner;
}

const counter = outer();
counter(); // logs 1
counter(); // logs 2
counter(); // logs 3
```

In this example, the `inner` function is a closure because it has access to the `count` variable from the `outer` function's scope, even after the `outer` function has returned.

## Understanding Scope

Scope refers to the region of the code where a variable is defined and can be accessed. In JavaScript, there are two main types of scope: global scope and local scope. Global scope refers to the scope of the entire program, while local scope refers to the scope of a specific function or block.

```javascript
let globalVariable = 'global';

function localScope() {
  let localVariable = 'local';
  console.log(globalVariable); // logs 'global'
  console.log(localVariable); // logs 'local'
}

localScope();
console.log(globalVariable); // logs 'global'
console.log(localVariable); // ReferenceError: localVariable is not defined
```

In this example, `globalVariable` is defined in the global scope and can be accessed from anywhere in the program. `localVariable`, on the other hand, is defined in the local scope of the `localScope` function and can only be accessed within that function.

## How Closures Interact with Scope

Closures can interact with scope in complex ways, and understanding these interactions is crucial for building robust applications. One important concept to understand is the idea of "variable hoisting". In JavaScript, variables are "hoisted" to the top of their scope, regardless of where they are actually defined. This means that a variable can be accessed before it is actually defined, which can lead to unexpected behavior.

```javascript
function outer() {
  console.log(inner); // logs undefined
  var inner = function() {
    console.log('inner');
  }
  console.log(inner); // logs the inner function
}

outer();
```

In this example, the `inner` variable is hoisted to the top of the `outer` function's scope, so it can be accessed before it is actually defined. However, because `inner` is a function, it is not initialized until the line where it is defined, so logging it before that line will result in `undefined`.

<Callout type="tip">
To avoid issues with variable hoisting, it's a good practice to define your variables at the top of their scope, and to use `let` and `const` instead of `var`.
</Callout>

## Real-World Applications of Closures and Scope

Closures and scope are not just theoretical concepts - they have real-world applications in building complex JavaScript applications. For example, closures can be used to create private variables and functions that are not accessible from outside a module, which is useful for building robust and secure applications.

```javascript
function createBankAccount(initialBalance) {
  let balance = initialBalance;

  function deposit(amount) {
    balance += amount;
  }

  function withdraw(amount) {
    balance -= amount;
  }

  function getBalance() {
    return balance;
  }

  return {
    deposit,
    withdraw,
    getBalance
  }
}

const account = createBankAccount(1000);
account.deposit(500);
account.withdraw(200);
console.log(account.getBalance()); // logs 1300
```

In this example, the `createBankAccount` function returns an object with `deposit`, `withdraw`, and `getBalance` methods. These methods are closures that have access to the `balance` variable, which is not accessible from outside the `createBankAccount` function. This provides a secure way to manage the account balance.
